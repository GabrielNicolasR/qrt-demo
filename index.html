O problema

Usuários abrem chamados no canal errado → triagem manual, atraso e custo. Nosso MVP usa checklist + texto do ticket para sugerir automaticamente o QRT com maior aderência (probabilidade).

Como funciona (resumo):

Recebe título/descrição e respostas do checklist → pré-processa texto + metadados → modelo (TF-IDF + XGBoost ou BERT) → predict_proba → exibe top‑3 QRTs com justificativas (palavras-chave + itens do checklist).
